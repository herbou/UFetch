## Project Overview

This is a Unity 6000.1.4f1 project using the Universal Render Pipeline (URP). The project includes a custom HTTP client wrapper (`UFetch`) built on top of Unity's `UnityWebRequest` system with Newtonsoft.Json integration for JSON serialization.

## Key Technologies

- **Unity Version**: 6000.1.4f1
- **Render Pipeline**: Universal Render Pipeline (URP) 17.1.0
- **JSON Library**: Newtonsoft.Json (via `jillejr.newtonsoft.json-for-unity`)
- **Input System**: New Unity Input System 1.14.0
- **Build System**: Unity's built-in build system (not using external build tools)

## Project Structure

- `Assets/Fetch/`: Custom HTTP client library (`UFetch.cs`) - a fetch-like API wrapper around UnityWebRequest
- `Assets/Scripts/`: Main game scripts (e.g., `Demo.cs` demonstrates UFetch usage)
- `Assets/Scenes/`: Unity scene files (`SampleScene.unity`)
- `Assets/Settings/`: URP rendering settings and volume profiles for PC and Mobile platforms
- `ProjectSettings/`: Unity project configuration files

## Key Packages

The project uses the following Unity packages:
- `com.unity.render-pipelines.universal` - Universal Render Pipeline
- `com.unity.inputsystem` - New Input System
- `com.unity.ai.navigation` - NavMesh navigation
- `com.unity.visualscripting` - Visual scripting tools
- `jillejr.newtonsoft.json-for-unity` - JSON serialization (installed via Git)

## UFetch HTTP Client

The `UFetch` class (in `Assets/Fetch/Fetch.cs`) provides a modern fetch-like API for HTTP requests in Unity:

- **Basic HTTP Methods**: `Get()`, `Post()`, `Put()`, `Patch()`, `Delete()`
- **JSON Helpers**: `PostJSON()`, `PutJSON()`, `PatchJSON()` with automatic serialization
- **Generic Typed Requests**: `Get<T>()`, `PostJSON<T>()` for automatic deserialization
- **Multipart Uploads**: `Multipart()` for file uploads with form fields
- **Progress Tracking**: Optional callbacks for download/upload progress
- **Response Object**: Includes `ok`, `status`, `text`, and `headers` properties

Example usage in `Assets/Scripts/Demo.cs` shows async/await patterns for network requests.

## Opening the Project

Open the project in Unity Hub by selecting the `EmptyForTesting` directory. Unity will automatically restore package dependencies from `Packages/manifest.json`.

## Building the Project

Unity generates C# project files automatically:
- `EmptyForTesting.sln` - Solution file for Visual Studio/Rider
- `Assembly-CSharp.csproj` - Runtime scripts assembly
- `Assembly-CSharp-Editor.csproj` - Editor scripts assembly

These files are regenerated by Unity when needed and should not be manually edited.

## Development Workflow

1. Open Unity Editor to work on scenes, prefabs, and assets
2. C# scripts in `Assets/` can be edited in your IDE of choice
3. Unity auto-compiles scripts on save
4. The project uses the new Input System - input actions are defined in `InputSystem_Actions.inputactions`
5. URP settings are split between PC and Mobile renderer configurations in `Assets/Settings/`

## Important Notes

- This project uses URP, not the built-in render pipeline - shaders and rendering features must be URP-compatible
- The project includes both PC and Mobile URP renderer assets for platform-specific optimizations
- HTTP requests should use the `UFetch` wrapper instead of raw `UnityWebRequest` for consistency
- Newtonsoft.Json is used for all JSON operations (not Unity's built-in JsonUtility)
